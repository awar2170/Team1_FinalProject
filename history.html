Data Exploration & Analysis

Data Selection

After researching several topics for data, we settled on mushrooms. The data available was great for machine learning applications. The source data is described as:

"Bookbased mushroom data set, describing physical features, binary classification in poisonous or edible. Data Set Description Files are reported for two data sets: Primary and Secondary. The primary data set encodes the textbook mushroom entries, while the secondary data set is a pilot data, result of simulation."

Data Preprocessing

In order to begin working with our dataset, we had to make a few adjustments to the CSV file. We first eliminated all null values from the primary and secondary dataset, including the placeholder “?” that was representing nulls in the dataset. Null values adversely affect the performance and accuracy of any machine learning algorithm. So, it is important to remove null values before applying a machine learning algorithm to the dataset. Next, we renamed the columns so the primary and secondary datasets could merge in a clean manner. Lastly, we dropped any columns that were not included in both datasets to integrate the two datasets.

Preliminary Feature Engineering and Feature Selection

We tested and compared multiple features for the input of the machine learning model. The inputs tested included RandomForest, TensorFlow, Correlation Analysis (using R), and XGBoost (img: 1, 2). We concluded XGBoost was the best option for our project and will allow the user to properly input the variables to predict if the mushroom is poisonous or edible.

Splitting the Data into Training and Testing Sets

The data was trained, tested, and split by using skelearn.model_selection and train_test_split functions. Training data is a portion of our actual dataset that is fed into the machine learning model to discover and learn patterns. Once the machine learning model is built with the training data, you need unseen data, or testing data, to test your model. Testing data is used to analyze the performance of the model’s training and will adjust it for the best results.

Training the model

The model was encoded using OneHotEncoder and train_test_split from sklearn. These functions allowed us to teach the model to classify certain objects according to their properties.

Accuracy Score

The accuracy score is used in classification problems to tell the percentage of accurate predictions. It is calculated by dividing the number of correct predictions by the total number of predictions.

The accuracy score for our model was 1.0 which tells us that our model is likely sufficient in predicting a correct outcome based on the given inputs.

Limitations and Benefits of using XGBoost

Advantages

  • Regularization which helps avoid overfitting
  • In-built cross-validation
  • Parallel Processing
  • Tree pruning using max_depth approach

Disadvantages

  • Sensitive to outliers
  • Poor performance on imbalanced data
  • Black-Box Nature is not always straightforward in explaining what is happening in the background
